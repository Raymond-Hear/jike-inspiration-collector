## 📌 项目背景

**核心价值**：从即刻（web.okjike.com）的深度内容分享中挖掘灵感和知识，作为Vibe Coding的选题来源。

**目标用户**：需要从即刻动态中收集、整理、扩展知识的创作者和开发者。

---

## 🎯 MVP 最小可行性产品

### 当前范围（Phase 1）

#### 核心功能 1：收集即刻灵感
**触发方式**：在即刻动态底部添加 **[💡 灵感]** 按钮

**交互流程**：
```
用户点击 [💡 灵感]
  ↓
弹出分类选择窗口
  ↓
用户选择类型 + 添加笔记
  ↓
保存到 chrome.storage.local
  ↓
显示"已收集"反馈
```

**分类类型**：
- [ ] **需求洞察**：动态中提到的痛点、问题、需求
- [ ] **干货分享**：文章感悟、视频笔记、技巧分享
- [ ] **洞见思考**：深度观点、独特视角

**保存的数据结构**：
```json
{
  "id": "unique_id",
  "type": "需求洞察 | 干货分享 | 洞见思考",
  "content": "动态正文内容",
  "author": "发布者昵称",
  "url": "动态链接",
  "note": "用户笔记",
  "collectedAt": "2025-01-07T12:00:00Z",
  "tags": []
}
```

---

#### 核心功能 2：本地存储和查看

**存储方式**：
- 使用 Chrome Extension Storage API
- 数据存储在用户本地浏览器
- 支持增删改查（CRUD）

**查看界面**：
- 插件 Popup（点击浏览器工具栏图标）
- 显示灵感列表（按时间倒序）
- 支持按类型筛选
- 支持搜索（笔记+内容）

---

#### 核心功能 3：导出功能

**导出格式**：
- [ ] JSON格式（完整数据，可备份）
- [ ] Markdown格式（可读性强，适合整理）

**导出内容**：
```markdown
# 即刻灵感收集

## 需求洞察 (3)

### 2025-01-07 - 标题
**来源**：https://web.okjike.com/...
**发布者**：用户A
**我的笔记**：这是一个很好的选题方向

**原始内容**：
[动态正文...]

---
```

## 📊 数据结构设计

### Inspiration Item
```typescript
interface InspirationItem {
  id: string;                    // 唯一标识
  type: 'need' | 'knowledge' | 'insight';  // 类型
  content: string;               // 动态正文
  author: string;                // 发布者
  authorUrl: string;             // 发布者主页
  url: string;                   // 动态链接
  note: string;                  // 用户笔记
  tags: string[];                // 标签
  collectedAt: string;           // 收集时间
  updatedAt: string;             // 更新时间
}
```

### Storage Schema
```typescript
interface StorageData {
  inspirations: InspirationItem[];
  settings: {
    defaultType: string;
    autoBackup: boolean;
  };
}
```

## 🎨 UI/UX 设计原则

### 设计理念
- **不打扰**：简洁的按钮，不破坏原页面体验
- **快速**：一键收集，最少点击次数
- **灵活**：支持后期编辑和分类

### 交互原则
1. 收集操作 < 3秒
2. 查找灵感 < 5秒
3. 导出数据 < 1次点击

## 📝 开发笔记

### 技术栈
- **前端**：Vanilla JS（无框架）
- **存储**：Chrome Storage API
- **样式**：CSS3（与即刻风格一致）
- **图标**：Emoji（💡）

### 关键技术点
1. **内容提取**：从即刻DOM结构提取动态信息
2. **按钮注入**：在合适位置添加[💡 灵感]按钮
3. **弹窗交互**：使用原生DOM操作创建弹窗
4. **数据持久化**：chrome.storage.local API
5. **导出功能**：生成Blob对象下载

### 已知挑战
- 即刻动态是异步加载，需要监听DOM变化
- 不同类型的动态（纯文字、图文、分享）DOM结构不同
- 需要处理动态路由变化（SPA页面）
